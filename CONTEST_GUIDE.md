# Agentic Player 10 공모전 가이드

## 1. 공모전 개요

- 주최: 카카오
- 후원: 과학기술정보통신부
- 예선: PlayMCP 서버 등록과 응모
- 본선: Kakao Tools 사용자 공개, 투표와 전문가 심사
- 본선 진출: 20개 서비스

## 2. 일정

| 단계 | 일정 |
|---|---|
| 예선 접수 | 2026-06-15 ~ 2026-07-14 |
| 본선 발표 | 2026-07-30 |
| 추가 개발 | 2026-07-30 ~ 2026-08-27 |
| 공개 투표 | 2026-08-31 ~ 2026-09-28 |
| 시상식 | 2026-10-23, 카카오 AI캠퍼스 |

PlayMCP 심사는 영업일 기준 최대 7일이다. 공식 안내상 7월 7일까지 요청된 심사는 7월 10일까지 처리 예정이며 이후 요청은 마감 전 완료되지 않을 수 있다.

## 3. 평가 기준

- 창의성: 새로운 방식으로 실제 문제를 해결하는가
- 편의성: 카카오톡에서 쉽게 이해하고 반복 사용할 수 있는가
- 안정성: 정확한 데이터, 안정적인 서버, 보안과 오류 처리가 있는가

AI My Health Manager 대응:

- 사진과 대화를 건강관리 행동으로 전환
- 사용자가 분류를 고르지 않는 단일 자유 입력
- 정부 공공데이터, 사용자 확인, 출처와 기준일 제공
- 건강 상담이 아니라 복약·병원 일정 이행 문제 해결

## 4. 필수 참가 절차

```text
MCP 서버 개발
→ PlayMCP in KC 서버 발급
→ Active 상태와 Endpoint 확인
→ PlayMCP 개발자 콘솔 임시 등록
→ 도구 선택·인자 바인딩 테스트
→ 등록 및 심사 요청
→ 승인 후 전체 공개
→ Player 예선 참여 제출
```

- PlayMCP 회원의 카카오 계정이 필요하다.
- PlayMCP in KC 서버는 계정당 최대 2대다.
- 서버는 공모전 참가 목적으로만 사용한다.
- 목적 외 사용하거나 예선에 접수하지 않으면 회수될 수 있다.
- 승인 직후 기본 공개 상태 `나에게만 공개`를 `전체 공개`로 변경한다.
- Player 예선 참여 제출은 1회만 가능하다.

## 5. PlayMCP in KC 필독 유의사항

- MCP 서버 발급 가능 기간은 예선 접수 기간인 `2026-06-15 ~ 2026-07-14`이다.
- Endpoint 발급 후 PlayMCP 등록과 심사를 진행해야 하므로 마감보다 충분히 일찍 발급한다.
- PlayMCP in KC에서 발급한 MCP Endpoint URL을 PlayMCP에 등록해야 참가할 수 있다.
- 공모전 참가작에 한해 한시적으로 무상 지원된다.
- 공모전 종료 후 일정 기간 유지된 뒤 무상 지원이 종료되며 정확한 종료일은 추후 공지를 확인한다.
- 무상 지원 종료 후 사업자는 별도 과금으로 유지하거나 다른 클라우드로 이전할 수 있다.
- 공모전 외 목적으로 사용하거나 예선에 접수하지 않으면 서버가 임의 회수될 수 있다.
- PlayMCP 회원만 이용할 수 있고 계정당 MCP 서버는 최대 2대다.

### 개발 지원과 문의

- 이 공모전은 참가자가 직접 MCP 서버를 개발하고 아이디어를 구현하는 방식이다.
- PlayMCP in KC 자체 오류 외에 MCP 개발 과정에서 발생하는 구현·디버깅 문제는 카카오가 별도로 해결하거나 지원하지 않는다.
- 개발 문제는 프로젝트 테스트, MCP Inspector, 공식 MCP SDK 문서와 로그를 이용해 자체 해결한다.
- 공모전 진행과 PlayMCP in KC 이용 문의는 [카카오 고객센터](https://cs.kakao.com/)로 문의한다.

## 6. 서버 운영

| 용도 | 이름 |
|---|---|
| 개발·임시 등록 | `ai-my-health-manager-dev` |
| 심사·공개 | `ai-my-health-manager-prod` |

개발 서버에서 검증하지 않은 빌드를 제출 서버에 배포하지 않는다. 심사와 공개 투표 기간에는 제출 서버를 임의로 중지하지 않는다.

## 7. AI와 MCP 규칙

- 공모전은 참가 MCP 서버의 특정 AI API 사용을 강제하지 않는다.
- PlayMCP/Kakao Tools AI 호스트가 도구 선택과 인자 구성을 담당한다.
- 참가 서버는 공공데이터 조회, 기록, 복약·병원 일정 같은 도구를 제공한다.
- 예선 기본 서버는 별도 생성형 AI API를 호출하지 않는다.
- 사진 전달 형태는 PlayMCP 임시 등록에서 직접 확인한다.
- `tools/list`에 노출되는 도구 설명과 입력 스키마를 정확히 작성한다.
- 정상 호출뿐 아니라 순차·병렬 호출과 오류 상황을 테스트한다.

## 8. 본선 준비

본선 진출 시 Kakao Tools 공개를 위한 추가 개발이 필수다.

- 강화된 MCP 표준 준수
- 확인·수정용 Widget
- 사용자 인증과 암호화 저장
- 사진 입력 UX 검증
- 복약·병원 일정 상태 UI
- 장애율, 확인 완료율, 일정 완료율 측정
- 개인정보 처리방침과 사용자 문의 채널

## 9. 권리와 데이터

- 서비스와 사용 데이터에 대한 적법한 권한이 있어야 한다.
- 제3자의 권리와 개인정보를 침해하지 않아야 한다.
- 테스트에는 가상·익명 건강 데이터를 사용한다.
- 공공데이터 이용조건과 출처표시를 준수한다.
- 실제 건강 사진과 처방전을 공개 저장소에 올리지 않는다.
