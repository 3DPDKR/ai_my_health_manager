# AI My Health Manager 프로그램 개발 가이드

## 1. 목표

사용자는 건강 항목이나 양식을 고르지 않고 사진 또는 대화를 그대로 입력한다. PlayMCP/Kakao Tools의 AI 호스트가 대화를 이해해 MCP 도구를 호출하면 서버가 건강 기록, 식단, 복약, 병원 일정으로 정리하고 공공데이터와 대조한다. 사용자 확인 후에만 저장·알림·추천으로 연결한다.

> 사진과 대화를 개인 건강 배경에 맞게 정리하고, 복약과 병원 일정을 실제로 관리하는 개인 건강관리 에이전트

## 2. 고정 기술 기준

| 항목 | 기준 |
|---|---|
| 프로젝트 | `AI My Health Manager` |
| 경로 | `C:\projects\ai_my_health_manager` |
| 서버명 | `ai-my-health-manager` |
| Python | 3.12 |
| MCP SDK | 공식 Python MCP SDK 1.x |
| Transport | Streamable HTTP |
| MCP | `/mcp` |
| Health | `/healthz` |
| 컨테이너 | `linux/amd64` |
| 시간대 | `Asia/Seoul` |

MCP 연결은 Stateless로 유지한다. 사용자 건강정보는 MCP 세션이나 컨테이너 로컬 파일이 아닌 외부 데이터베이스에 저장한다.

## 3. AI 사용 원칙

```text
사용자 사진·대화
→ PlayMCP/Kakao Tools AI 호스트
→ Tool Selection과 Argument Binding
→ AI My Health Manager MCP
→ 공공데이터 조회·상태 처리
→ 구조화 결과
→ AI 호스트 최종 답변
```

- 공모전은 참가 서버의 특정 생성형 AI API 사용을 요구하지 않는다.
- 예선 서버는 별도 OpenAI·Gemini·Claude API를 호출하지 않는다.
- AI 호스트는 자연어 이해, 도구 선택, 인자 구성과 최종 답변을 담당한다.
- MCP 서버는 입력 검증, 공공데이터, 기록, 복약·일정 상태와 안전 규칙을 담당한다.
- 사진이 도구에 원본·파일 참조·설명 중 어떤 형태로 전달되는지는 PlayMCP 임시 등록에서 확인한다.
- 별도 AI API는 공모전 허용, 개인정보 동의, 비밀키 관리, 비용·장애 대책을 갖춘 뒤에만 검토한다.

## 4. 핵심 흐름

```text
사진 또는 대화
→ 응급 표현 확인
→ 시각·내용·대화 맥락 분석
→ 여러 건강 항목으로 분리
→ 공공데이터 후보 매칭
→ 한 번의 짧은 확인
→ 승인된 항목만 저장
→ 복약·병원 일정·생활관리 반영
```

사용자는 `응`, `혈압은 128/84야`, `약은 아직 안 먹었어`처럼 자연어로 승인·수정·거절한다.

## 5. 데이터 분류

| 분류 | 내용 |
|---|---|
| `vitals` | 혈압, 맥박, 체온, 혈당, 산소포화도, 호흡수 |
| `body` | 키, 체중, 허리둘레, BMI |
| `meal` | 음식, 영양, 수분, 식사 시각 |
| `medication` | 약, 용량, 횟수, 복용법, 복용 완료 |
| `symptom` | 통증, 두통, 어지러움, 흉통, 호흡, 소화, 피부 |
| `activity` | 걷기, 운동, 수면 |
| `medical_history` | 질환, 만성질환, 알레르기, 수술·입원 |
| `document` | 처방전, 검사 결과, 진단서, 퇴원 안내문 |
| `hospital_schedule` | 방문, 검사, 재진, 준비사항 |
| `prevention` | 검진, 예방접종, 치과·안과 일정 |
| `unknown` | 추가 설명 필요 |

각 기록은 발생 시각, 값, 단위, 출처, 신뢰도, 확인 상태, 수정 시각, 삭제 상태를 가진다.

## 6. 사진 처리

| 사진 | 추출 후보 | 처리 |
|---|---|---|
| 음식 | 음식명, 양, 식사 시각 | 영양 DB 매칭 |
| 약·약봉투 | 제품명, 함량, 횟수, 복용법 | 식약처 매칭 |
| 처방전 | 처방일, 약제, 복용법, 기간 | 복약 계획 |
| 검사표 | 검사명, 결과, 단위, 기준범위 | 변화 정리 |
| 혈압·혈당계 | 수치와 단위 | 활력징후 |
| 진단서·퇴원 안내 | 진료일, 재진 조건 | 병원 일정 |
| 운동 화면 | 운동, 시간, 거리, 걸음 | 활동 기록 |

시간은 사용자 명시 시각 → 안전하게 전달된 촬영 시각 → 파일 생성 시각 → 업로드 시각 순으로 판단한다. 충돌하면 확인한다. EXIF 위치정보는 사용하지 않는다. CT, MRI, X-ray는 판독하지 않는다.

## 7. 정부 공공데이터

| 분야 | 데이터 |
|---|---|
| 의약품 | 식약처 제품 허가정보 `DrugPrdtPrmsnInfoService07` |
| 복약 안내 | 식약처 e약은요 |
| 알약 후보 | 식약처 낱알식별 `MdcinGrnIdntfcInfoService03` |
| 약품 주의 | 식약처 DUR 성분·품목정보 |
| 식단 | 전국통합식품영양성분정보 `15100064` |
| 병원 | 심평원 병원정보서비스 |
| 건강 안내 | 질병관리청 국가건강정보포털 |

규칙:

- 공공데이터에 없으면 AI가 값을 만들지 않는다.
- 약은 품목코드 우선, 없으면 제품명·함량·제형·제조사를 비교한다.
- 알약 사진만으로 제품을 확정하지 않는다.
- DUR 결과는 의료진·약사 확인 항목이며 약 중단 지시가 아니다.
- 음식 중량은 범위로 추정하고 사용자가 확인한다.
- 영양값에는 기준량·제공기관·기준일을 표시한다.
- 병원 API는 실제 예약 가능 시간이나 휴진을 보장하지 않는다.

외부 결과에는 `provider`, `dataset`, `source_record_id`, `retrieved_at`, `effective_date`, `match_method`, `confidence`를 저장한다.

## 8. MCP 도구

| 도구 | 역할 |
|---|---|
| `organize_health_input` | 대화와 사진 설명을 저장 전 후보로 분리 |
| `confirm_health_observations` | 승인·수정·거절 |
| `get_health_context` | 관련된 최소 건강 배경 조회 |
| `lookup_medication` | 제품·낱알·e약은요 검색 |
| `check_medication_cautions` | DUR 주의정보 |
| `lookup_food_nutrition` | 식품영양 검색·환산 |
| `update_medication_status` | 복약 완료·누락 변경 |
| `manage_medication_plan` | 복약 일정 관리 |
| `manage_hospital_schedule` | 병원·검사·재진 일정 |
| `get_health_summary` | 확인된 최근 기록 요약 |
| `recommend_next_action` | 생활관리 다음 행동 |
| `check_emergency_signals` | 응급 의심 표현 확인 |

도구 설명에는 호출 시점과 금지 행동을 함께 적는다. 오류도 JSON으로 반환해 AI 호스트가 재질문할 수 있게 한다.

## 10. 알림과 추천

- 확인된 처방과 일정으로만 알림을 만든다.
- 복약은 확인된 시각 또는 식후 지연 시간을 사용한다.
- 병원 기본 알림은 7일 전, 3일 전, 1일 전, 당일이다.
- 다음 일정은 실제 방문 완료일 기준이며 주말·확인된 공휴일이면 이전 평일을 제안한다.
- 실제 메시지 전달은 Kakao Tools가 지원하는 범위에서만 제공한다.
- 추천은 식단 과다·부족, 생활 패턴, 목표에 맞춘 다음 행동, 일정 누락 방지, 진료 준비로 제한한다.
- 진단, 약 변경, 치료 결정, 근거 없는 의약품·건강식품 추천은 금지한다.

## 11. 안전과 개인정보

- 의료기기가 아니며 진단·치료·예방을 대신하지 않는다.
- 흉통, 호흡곤란, 의식 이상, 얼굴 처짐, 말 어눌함, 한쪽 힘 빠짐, 갑작스러운 심한 두통·시야 이상은 먼저 안내한다.
- 자동 전화·문자·신고를 하지 않는다.
- 미확인 결과는 24시간 후 삭제한다.
- 원본 사진은 확인 후 삭제가 기본이다.
- 확인된 기록은 암호화 저장하고 삭제 기능을 제공한다.
- API 키, 인증 토큰, 원문 건강정보를 로그에 남기지 않는다.
