# AI 건강비서 앱 재구축 기준

## 1. 프로젝트 원칙

- 기존 Flutter 앱 코드, 임시 패치, 실패한 워크플로 구조는 재사용하지 않는다.
- MCP, PlayMCP, 카카오 공모전 관련 코드·문서·문구는 포함하지 않는다.
- 앱에는 사용자의 병원 일정, 질환, 복약 정보, 샘플 건강 데이터를 기본 등록하지 않는다.
- 사용자가 대화·사진·촬영·음성으로 입력하고 확인한 정보만 저장한다.
- 기능을 한 번에 모두 넣지 않고, 단계별로 빌드·실행 확인 후 다음 기능을 추가한다.

## 2. 핵심 사용자 경험

앱의 첫 화면은 대시보드 카드가 아니라 `AI 건강비서 대화창`이다.

사용자는 메뉴를 먼저 선택하지 않고 아래 방식으로 건강 정보를 입력한다.

- 텍스트 채팅
- 갤러리 사진 첨부
- 카메라 촬영
- 음성 입력
- 여러 사진 동시 입력: 최대 5장

AI는 입력 내용을 분석하여 데이터 유형과 사용자의 의도를 판단한다.

처리 흐름:

1. 사용자 입력
2. Gemini 분석
3. 데이터 분류 및 구조화
4. 저장 후보 표시
5. 사용자가 수정·저장·취소
6. 확인된 항목만 로컬 DB에 저장
7. 이후 AI 대화로 검색·조회·요약

## 3. AI 데이터 분류

- 활력징후: 혈당, 혈압, 맥박, 체온, 산소포화도
- 신체정보: 체중, BMI
- 식사: 음식, 수분, 식사 시각
- 복약: 약 이름, 용량, 횟수, 복용 시각, 복약 완료
- 증상: 증상명, 시작 시각, 정도, 변화
- 활동: 운동, 수면
- 병력: 질환, 입원, 수술
- 문서: 처방전, 약봉투, 검사 결과표, 진단서, 퇴원 안내
- 일정: 병원, 검사, 재진, 복약 일정
- 기타: 분류 불가 또는 사용자 질문

## 4. AI 의도 분류

- 기록 저장
- 기록 조회
- 기록 수정
- 기록 삭제
- 복약 완료 처리
- 병원 일정 저장
- 문서 또는 사진 분석
- 통계 및 요약 요청
- 일반 건강 질문

## 5. Gemini API

- 앱 설정 화면에서 Gemini API 키를 직접 입력할 수 있어야 한다.
- API 키는 Android 보안 저장소에 저장한다.
- 화면에는 키 원문을 노출하지 않고 일부만 가린다.
- API 연결 테스트 버튼을 제공한다.
- API 오류 시 사용자가 이해할 수 있는 메시지를 표시한다.
- 네트워크 오류, HTTP 상태 코드, 응답 파싱 오류를 구분한다.
- API 키 원문은 오류 로그에 기록하지 않는다.

## 6. 다국어

기본값은 Android 시스템 언어를 따른다.

설정에서 아래 언어를 직접 선택할 수 있다.

- 시스템 설정 따름
- 한국어
- English
- 日本語
- 简体中文

선택한 언어는 다음에 적용한다.

- 앱 UI
- 음성 인식 locale
- AI 응답 언어

선택값은 앱 재실행 후에도 유지한다.

## 7. 사진 입력

- 갤러리 다중 선택: 최대 5장
- 카메라로 사진 추가
- 선택 사진 미리보기
- 개별 삭제
- 한 번에 하나의 자료 묶음으로 분석
- 일부 사진 실패 시 전체 앱이 종료되지 않도록 처리
- 전송 전 이미지 크기 축소

예시 묶음:

- 처방전 전체
- 약봉투 앞면
- 약봉투 뒷면
- 약 상자
- 복약 안내문

AI는 사진별 결과와 통합 결과를 구분한다.

## 8. 저장 및 불러오기

- AI 분석 결과를 곧바로 저장하지 않는다.
- 사용자 확인 후 저장한다.
- 저장된 데이터는 카테고리, 요약, 원본 입력 유형, 발생 시각, 생성 시각을 가진다.
- 사용자는 AI 대화로 저장 데이터를 불러온다.

예:

- 최근 혈당 보여줘
- 이번 달 병원 일정
- 내가 먹는 약
- 지난 검사 결과 요약
- 혈압 변화 알려줘

## 9. 오류 로그

설정에 `오류 로그` 메뉴를 둔다.

기록 항목:

- 발생 시각
- 앱 버전
- Android 버전
- 작업 단계
- 오류 유형
- 안전하게 가린 오류 메시지
- HTTP 상태 코드가 있으면 상태 코드

기능:

- 최근 오류 보기
- 전체 로그 복사
- 로그 파일 공유
- AI 대화창으로 보내 분석
- 로그 전체 삭제

로그에 저장하지 않는 정보:

- API 키 원문
- 건강 대화 전문
- 처방전·검사표 원본 이미지
- 주민번호, 환자번호, 인증 토큰

## 10. 화면 구성

하단 메뉴는 다음 4개로 제한한다.

1. AI 건강비서
2. 저장 기록
3. 일정
4. 설정

`AI 건강비서`가 기본 화면이다.

대화창 하단 입력 영역:

- 사진/첨부 버튼
- 음성 버튼
- 텍스트 입력창
- 전송 버튼

## 11. 개발 및 빌드 순서

### 1단계

- 새 Flutter 프로젝트 생성
- 앱 실행
- 메인 대화 UI
- 설정 화면
- 다국어 전환
- Release APK 빌드

### 2단계

- API 키 보안 저장
- Gemini 텍스트 대화
- API 연결 테스트
- 오류 로그
- Release APK 빌드

### 3단계

- 사진 최대 5장
- 카메라
- 음성 입력
- Gemini 멀티모달 분석
- Release APK 빌드

### 4단계

- AI 데이터 분류
- 저장 확인 화면
- 로컬 DB 저장
- AI 기반 조회·수정·삭제
- Release APK 빌드

각 단계에서 반드시 다음을 통과한 뒤 다음 단계로 진행한다.

- `flutter pub get`
- `flutter analyze`
- 기본 위젯 테스트
- `flutter build apk --release`
- 실제 Android 기기 실행 확인

## 12. 금지 사항

- 빌드 워크플로에서 앱 소스 코드를 문자열 치환으로 수정하지 않는다.
- YAML 안에 Dart 코드 패치를 삽입하지 않는다.
- 기본 생성 테스트가 현재 앱 클래스와 맞지 않는 상태로 두지 않는다.
- 경고를 숨겨 오류를 통과시키지 않는다.
- 빌드 성공을 확인하지 않은 APK를 제공하지 않는다.
- 기존 실패 코드에 계속 덧붙이는 방식으로 수정하지 않는다.
