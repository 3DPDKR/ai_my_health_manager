# AI 건강비서 앱 재구축 기준

## 1. 프로젝트 원칙

- 기존 Flutter 앱 코드, 임시 패치, 실패한 워크플로 구조는 재사용하지 않는다.
- MCP, PlayMCP, 카카오 공모전 관련 코드·문서·문구는 포함하지 않는다.
- 앱에는 사용자의 병원 일정, 질환, 복약 정보, 샘플 건강 데이터를 기본 등록하지 않는다.
- 사용자가 대화·사진·촬영·음성으로 입력하고 확인한 정보만 저장한다.
- 기능을 한 번에 모두 넣지 않고, 단계별로 빌드·실행 확인 후 다음 기능을 추가한다.

## 2. 핵심 사용자 경험

앱의 첫 화면은 대시보드 카드가 아니라 `AI 건강비서 대화창`이다.

사용자는 메뉴를 먼저 선택하지 않고 아래 방식으로 건강 정보를 입력한다.

- 텍스트 채팅
- 갤러리 사진 첨부
- 카메라 촬영
- 음성 입력
- 여러 사진 동시 입력: 최대 5장

AI는 입력 내용을 분석하여 데이터 유형과 사용자의 의도를 판단한다.

처리 흐름:

1. 사용자 입력
2. Gemini 분석
3. 데이터 분류 및 구조화
4. 저장 후보 표시
5. 사용자가 수정·저장·취소
6. 확인된 항목만 로컬 DB에 저장
7. 이후 AI 대화로 검색·조회·요약

## 3. AI 데이터 분류

- 활력징후: 혈당, 혈압, 맥박, 체온, 산소포화도
- 신체정보: 체중, BMI
- 식사: 음식, 수분, 식사 시각
- 복약: 약 이름, 용량, 횟수, 복용 시각, 복약 완료
- 증상: 증상명, 시작 시각, 정도, 변화
- 활동: 운동, 수면
- 병력: 질환, 입원, 수술
- 문서: 처방전, 약봉투, 검사 결과표, 진단서, 퇴원 안내
- 일정: 병원, 검사, 재진, 복약 일정
- 기타: 분류 불가 또는 사용자 질문

## 4. AI 의도 분류

- 기록 저장
- 기록 조회
- 기록 수정
- 기록 삭제
- 복약 완료 처리
- 병원 일정 저장
- 문서 또는 사진 분석
- 통계 및 요약 요청
- 일반 건강 질문

## 5. Gemini API

- 앱 설정 화면에서 Gemini API 키를 직접 입력할 수 있어야 한다.
- API 키는 Android 보안 저장소에 저장한다.
- 화면에는 키 원문을 노출하지 않고 일부만 가린다.
- API 연결 테스트 버튼을 제공한다.
- API 오류 시 사용자가 이해할 수 있는 메시지를 표시한다.
- 네트워크 오류, HTTP 상태 코드, 응답 파싱 오류를 구분한다.
- API 키 원문은 오류 로그에 기록하지 않는다.

## 6. 다국어

기본값은 Android 시스템 언어를 따른다.

설정에서 아래 언어를 직접 선택할 수 있다.

- 시스템 설정 따름
- 한국어
- English
- 日本語
- 简体中文

선택한 언어는 다음에 적용한다.

- 앱 UI
- 음성 인식 locale
- AI 응답 언어

선택값은 앱 재실행 후에도 유지한다.

## 7. 사진 입력

- 갤러리 다중 선택: 최대 5장
- 카메라 연속 촬영
- 선택한 사진의 순서 변경
- 개별 사진 삭제
- 개별 사진 다시 촬영
- 사진별 확대 보기
- 선택 사진 미리보기
- 한 번에 하나의 자료 묶음으로 Gemini에 전송하여 종합 분석
- 일부 사진 실패 시 전체 앱이 종료되지 않도록 처리
- 전송 전 이미지 크기 축소

예시 묶음:

- 처방전 전체
- 약봉투 앞면
- 약봉투 뒷면
- 약 상자
- 복약 안내문

AI는 사진별 결과와 통합 결과를 구분한다.

## 8. 데이터 저장·분류·복원·불러오기

### 8.1 저장 원칙

- AI 분석 결과를 곧바로 저장하지 않는다.
- 사용자가 저장 후보를 확인하고 수정·저장·취소 중 하나를 선택한다.
- 확인된 항목만 로컬 DB에 저장한다.
- 저장 시 원본 입력 전체가 아니라 구조화된 데이터와 필요한 최소 메타데이터를 저장한다.
- 원본 사진은 사용자가 별도 보관을 선택한 경우에만 앱 저장소에 보관한다.

### 8.2 공통 저장 필드

모든 건강 데이터는 최소한 아래 필드를 가진다.

- 고유 ID
- 카테고리
- 세부 유형
- 제목 또는 요약
- 구조화된 값
- 단위
- 발생 시각
- 생성 시각
- 수정 시각
- 입력 방식: 채팅, 음성, 사진, 카메라, OCR, 문서
- AI 신뢰도
- 사용자 확인 여부
- 원본 참조 ID
- 삭제 여부

### 8.3 카테고리별 저장

- 활력징후: 혈당, 혈압, 맥박, 체온, 산소포화도
- 신체정보: 체중, BMI
- 복약: 약 이름, 용량, 복용 횟수, 복용 시간, 시작일, 종료일, 복약 완료
- 일정: 병원명, 진료과, 날짜, 시간, 검사·재진 여부
- 증상: 증상명, 위치, 정도, 시작 시각, 지속 시간, 변화
- 식사: 음식명, 식사 시각, 영양 추정값, 사진 참조
- 활동: 운동 종류, 시간, 거리, 수면 시간
- 병력: 질환, 입원, 수술, 진단 시점
- 문서: 처방전, 검사표, 진단서, 퇴원 안내의 구조화 결과

### 8.4 AI 자동 분류

- AI는 입력 내용을 한 개 이상의 카테고리로 분류할 수 있다.
- 한 문장이나 사진 묶음에서 여러 기록 후보를 생성할 수 있다.
- 중복 가능성이 있으면 기존 데이터와 비교하여 사용자에게 병합 여부를 묻는다.
- 날짜·시간·단위가 불명확하면 추정값으로 표시하고 저장 전 확인을 요청한다.
- AI가 확신하지 못하는 항목은 `확인 필요` 상태로 저장 후보에 표시한다.

### 8.5 데이터 불러오기

사용자는 AI 대화로 저장 데이터를 불러온다.

예:

- 최근 혈당 보여줘
- 이번 달 병원 일정
- 내가 먹는 약
- 지난 검사 결과 요약
- 혈압 변화 알려줘
- 지난주에 기록한 증상 보여줘
- 어제 저장한 사진 분석 결과 다시 보여줘

불러오기 시 지원 기능:

- 기간 검색
- 카테고리 검색
- 키워드 검색
- 최신순·오래된순 정렬
- 평균·최소·최대·변화량 계산
- 기록 원문 요약 보기
- 관련 일정과 복약 정보 함께 보기

### 8.6 수정과 삭제

- 사용자는 AI 대화로 저장 데이터를 수정할 수 있다.
- 수정 전 기존값과 변경값을 비교해서 보여준다.
- 삭제는 즉시 완전 삭제하지 않고 휴지통 상태로 이동한다.
- 휴지통 데이터는 일정 기간 복원할 수 있다.
- 영구 삭제는 별도 확인 후 실행한다.

### 8.7 백업과 복원

설정에 `데이터 백업·복원` 메뉴를 둔다.

백업 대상:

- 건강 기록
- 복약 정보
- 병원·검사 일정
- 저장된 AI 분석 결과
- 사용자 설정
- 언어 설정
- 카테고리 설정

백업 제외 대상:

- Gemini API 키 원문
- 임시 캐시
- 전송 대기 이미지
- 오류 로그에 포함된 민감정보

백업 형식:

- 앱 전용 JSON 또는 ZIP
- 스키마 버전 포함
- 생성 시각 포함
- 데이터 개수 요약 포함
- 선택적으로 암호 설정 가능

복원 절차:

1. 백업 파일 선택
2. 파일 무결성 및 스키마 버전 검사
3. 복원 예정 데이터 개수 표시
4. 기존 데이터와 중복 검사
5. 덮어쓰기·병합·취소 선택
6. 복원 실행
7. 복원 결과와 실패 항목 표시

복원 안전 원칙:

- 복원 전 현재 데이터를 자동 임시 백업한다.
- 복원 실패 시 기존 상태로 되돌릴 수 있어야 한다.
- 일부 항목만 실패해도 성공한 데이터와 실패한 데이터를 구분해 보여준다.
- 앱 버전이 달라도 가능한 범위에서 데이터 마이그레이션을 수행한다.

### 8.8 로컬 DB와 데이터 계층

- 로컬 DB는 구조화된 건강 기록 저장을 기준으로 설계한다.
- UI에서 DB에 직접 접근하지 않는다.
- Repository 계층을 통해 저장·조회·수정·삭제·복원을 수행한다.
- AI 응답과 DB 모델을 분리한다.
- AI가 반환한 JSON은 검증 후 도메인 모델로 변환한다.
- 스키마 버전을 관리하고 마이그레이션 테스트를 둔다.

## 9. 오류 로그

설정에 `오류 로그` 메뉴를 둔다.

기록 항목:

- 발생 시각
- 앱 버전
- Android 버전
- 작업 단계
- 오류 유형
- 안전하게 가린 오류 메시지
- HTTP 상태 코드가 있으면 상태 코드

기능:

- 최근 오류 보기
- 전체 로그 복사
- 로그 파일 공유
- AI 대화창으로 보내 분석
- 로그 전체 삭제

로그에 저장하지 않는 정보:

- API 키 원문
- 건강 대화 전문
- 처방전·검사표 원본 이미지
- 주민번호, 환자번호, 인증 토큰

## 10. 화면 구성

하단 메뉴는 다음 4개로 제한한다.

1. AI 건강비서
2. 저장 기록
3. 일정
4. 설정

`AI 건강비서`가 기본 화면이다.

대화창 하단 입력 영역:

- 사진/첨부 버튼
- 음성 버튼
- 텍스트 입력창
- 전송 버튼

## 12. 금지 사항

- 빌드 워크플로에서 앱 소스 코드를 문자열 치환으로 수정하지 않는다.
- YAML 안에 Dart 코드 패치를 삽입하지 않는다.
- 기본 생성 테스트가 현재 앱 클래스와 맞지 않는 상태로 두지 않는다.
- 경고를 숨겨 오류를 통과시키지 않는다.
- 빌드 성공을 확인하지 않은 APK를 제공하지 않는다.
- 기존 실패 코드에 계속 덧붙이는 방식으로 수정하지 않는다.
