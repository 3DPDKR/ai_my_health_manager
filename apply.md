# AGENTIC PLAYER 10 참가 신청서

아래 `제출 문안`의 코드 블록 안에 있는 내용만 Kakao 비즈니스폼에 복사한다.

## 제출 문안

### 이름

```text
강대현
```

### [신청1] PlayMCP 서비스명

```text
ai_my_health_manager
```

### [신청1] PlayMCP 상세 페이지 URL 주소

```text
https://playmcp.kakao.com/mcp/[실제 서비스 ID]
```

> 제출 전 PlayMCP에서 서비스를 전체 공개한 뒤, 공개 상세 페이지의 실제 URL로 반드시 교체한다. Kakao Cloud의 MCP Endpoint 주소를 입력하면 안 된다.

### [신청2] PlayMCP 서비스명 (선택)

입력하지 않는다.

### [신청2] PlayMCP 상세 페이지 URL 주소 (선택)

입력하지 않는다.

### 서비스 소개 및 AGENTIC PLAYER 10 지원 사유

```text
AI My Health Manager는 대화와 사진 등 다양한 입력을 건강 기록으로 구조화하고, 복약과 병원 일정을 실제 관리 행동으로 연결하는 개인 건강관리 MCP 서비스입니다.

건강정보는 혈압계 화면, 약봉투, 처방전, 메모와 캘린더처럼 여러 형태와 장소에 흩어져 있습니다. 사용자가 매번 앱의 항목과 양식을 찾아 입력해야 하므로 기록을 꾸준히 유지하기 어렵고, 진료 시점에는 중요한 변화와 복약 이력을 다시 떠올려야 합니다. AI My Health Manager는 사용자가 “오늘 혈압은 132/86이고 저녁 약은 먹었어”처럼 평소 말투로 입력하면 측정값, 증상, 복약 여부와 병원 일정을 구분해 확인 가능한 기록 초안으로 정리합니다.

현재 MVP는 자연어 입력을 기반으로 응급 의심 표현 확인, 여러 건강 항목의 기록 초안 생성, 확인된 처방에 따른 복약 알림 시각 계산, 실제 방문 완료일 기준의 다음 진료일 계산, 의료진에게 보여줄 방문 브리핑 생성을 제공합니다. 카카오의 대화형 AI가 상황에 맞는 MCP 도구를 선택하고 순차적으로 호출하므로 사용자는 복잡한 메뉴를 배우지 않고 대화만으로 관리 흐름을 이어갈 수 있습니다.

핵심 차별점은 질문에 답하고 끝나는 건강 상담 챗봇이 아니라 다음의 ‘확인 기반 실행 루프’를 제공한다는 점입니다.

대화·사진 입력 → 응급 신호 우선 확인 → 건강 기록 후보 분리 → 사용자 확인·수정 → 복약·진료 행동 연결 → 다음 대화에서 이행 점검

의료 영역에서는 편의성보다 안전성을 먼저 설계했습니다. AI가 입력 내용을 임의로 확정하지 않도록 모든 추출 결과와 일정은 초안으로 반환하며 사용자 확인 전에는 저장하거나 실행하지 않습니다. 진단, 치료 결정과 처방 변경을 하지 않고, 응급 가능성이 있으면 119 또는 의료기관의 도움을 우선 안내하되 자동 신고나 연락은 수행하지 않습니다.

AGENTIC PLAYER 10 본선에서는 약봉투·처방전·측정기 사진의 인식 및 확인 흐름, 식약처 의약품과 공공 식품영양 데이터 연동, 사용자 인증과 암호화 저장, Kakao Tools 확인·수정 Widget을 구현하겠습니다. 또한 도구 선택 정확도, 기록 확인 완료율, 복약·일정 행동 완료율을 측정해 실질적인 사용자 가치를 검증하겠습니다.

사용자가 건강관리를 위해 새로운 앱 사용법을 배우는 것이 아니라, 카카오톡에서 말하거나 사진을 보내면 에이전트가 안전하게 기록하고 다음 행동까지 놓치지 않게 돕는 것. 이것이 AI My Health Manager가 만들고자 하는 변화이며 AGENTIC PLAYER 10에 지원하는 이유입니다.
```

## 심사 기준 대응 요약

이 부분은 제출 문안의 논리를 점검하기 위한 메모이며 비즈니스폼에 복사하지 않는다.

| 평가 기준 | 강조한 내용 |
|---|---|
| 창의성 | 답변형 건강 챗봇이 아닌 `확인 기반 실행 루프`와 여러 MCP 도구의 순차 조합 |
| 편의성 | 항목 선택 없이 대화·사진으로 기록·복약·진료 관리, 카카오 환경에서 반복 사용 |
| 안정성 | 응급 신호 우선 확인, 사용자 승인 전 미확정, 진단·처방 변경 금지, 구조화된 오류 처리 |
| 파급력 | 고령자·만성질환자·가족 돌봄에서 시작해 일상 건강관리 사용자로 확장 |
| 본선 실행력 | 사진 입력, 공공데이터, 인증·암호화 저장, Widget, 정량 지표의 구체적인 개발 계획 |

## 제출 전 필수 확인

- [ ] PlayMCP 심사를 통과했다.
- [ ] 공개 상태를 `나에게만 공개`에서 `전체 공개`로 변경했다.
- [ ] 실제 PlayMCP 공개 상세 페이지 URL로 교체했다.
- [ ] URL이 로그인하지 않은 브라우저에서도 열리는지 확인했다.
- [ ] 공개 서비스명과 신청서의 `ai_my_health_manager`가 일치한다.
- [ ] PlayMCP에서 핵심 도구를 직접 호출해 정상 응답을 확인했다.
- [ ] 신청2의 두 항목은 빈칸으로 두었다.
- [ ] 최종 제출은 1회만 가능하므로 모든 항목을 다시 확인했다.
