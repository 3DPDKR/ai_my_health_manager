# AGENTIC PLAYER 10 참가 신청서 최종 입력안

폼의 각 항목에 아래 값을 그대로 입력한다. 코드 블록 밖의 안내문은 입력하지 않는다.

## 1. 이름 — 3/200자

```text
강대현
```

## 2. [신청1] PlayMCP 서비스명 — 20/200자

```text
ai_my_health_manager
```

## 3. [신청1] PlayMCP 상세 페이지 URL 주소

```text
https://playmcp.kakao.com/mcp/[실제 서비스 ID]
```

중요:

- 이 칸에 서비스 소개 문장을 입력하지 않는다.
- PlayMCP 심사 승인 후 서비스를 `전체 공개`로 변경한다.
- 공개 상세 페이지에서 실제 URL을 복사해 위 예시 전체를 교체한다.
- `[실제 서비스 ID]`가 남은 상태로 제출하지 않는다.
- Kakao Cloud의 `.../mcp` Endpoint 주소가 아니라 PlayMCP 공개 상세 페이지 URL을 입력한다.

## 6. 서비스 소개 및 AGENTIC PLAYER 10 지원 사유 — 187/200자

```text
AI My Health Manager는 대화로 혈압·증상·복약·진료 일정을 기록 초안으로 정리하고 응급 신호를 우선 확인하는 MCP 서비스입니다. 복잡한 입력 없이 건강관리를 행동으로 연결합니다. AGENTIC PLAYER 10에서 도구 선택 정확도와 확인 절차를 검증하고, 본선에서 사진 입력·공공데이터·확인 위젯으로 확장하겠습니다.
```

## 7. PlayMCP 서비스를 `전체 공개` 상태로 설정하셨나요?

PlayMCP 심사 승인 후 실제로 전체 공개를 설정하고 공개 페이지 접속까지 확인한 경우에만 선택한다.

```text
네, '전체 공개' 설정 상태입니다. (미설정시 참여 불가)
```

현재 서비스가 심사 승인·전체 공개 전이라면 아직 이 항목을 선택하거나 참가 신청을 제출하지 않는다.

## 제출 전 필수 체크리스트

- [ ] GitHub 저장소에 최신 프로그램이 공개되어 있다.
- [ ] Kakao Cloud MCP 서버 상태가 `Active`다.
- [ ] PlayMCP `정보 불러오기`에서 5개 도구가 모두 표시된다.
- [ ] PlayMCP 임시 등록 상태에서 정상·응급·오류 테스트를 완료했다.
- [ ] `등록 및 심사 요청`이 승인되었다.
- [ ] 공개 상태를 `나에게만 공개`에서 `전체 공개`로 변경했다.
- [ ] 로그아웃 상태에서 PlayMCP 공개 상세 페이지가 열린다.
- [ ] 신청1 URL 칸에 서비스 설명이 아닌 실제 공개 상세 페이지 URL을 입력했다.
- [ ] 신청2의 두 항목은 비워 두었다.
- [ ] 서비스 소개 문안이 200자 이하인지 확인했다.
- [ ] 소속 구분은 `개인개발/프리랜서`, 소속명은 `해당 없음`으로 입력했다.
- [ ] 최종 제출은 1회만 가능하므로 전체 내용을 다시 검토했다.
